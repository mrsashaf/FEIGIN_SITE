# Project Map (Current)

## Root pages

- `index.html` - main landing page with hero, lore, cases, archive, clearance.
- `analyst.html` - case page: CASE 01.
- `censored.html` - case page: CASE 02.
- `design-system.html` - visual reference of components.
- `admin.html` - visual editor utility page.

## CSS

- `css/shared.css` - reset + shared header/footer/buttons + shared responsive behavior.
- `css/index.css` - main page sections and interactions styling.
- `css/cases.css` - case page variants and shared case components.

## JavaScript

- `js/evidence-filters.js` - evidence filters, load more, clearance form state transitions.
- `js/access-system.js` - access checks, warnings, preview modal, typewriter subtitle.
- `js/case-slideshow.js` - CASE 01 image slideshow behavior.
- `js/page-transitions.js` - fade transitions + smooth anchor scrolling.

## Assets

- `assets/brand/` - logos and brand overlays.
- `assets/index/` - home page hero/prism media.
- `assets/cases/` - case-specific media.
- `assets/evidence/` - evidence and placeholders.

## Supporting files

- `CLAUDE.md` - full design/implementation instruction set for AI.
- `docs/AI_GUARDRAILS.md` - concise execution contract for day-to-day changes.
- `docs/SEMANTIC_ROLE_MAP.md` - canonical mapping of H1/H2/H3, buttons, links, and cards.
- `docs/DESIGN_RULES_PLAYBOOK.md` - protocol for consistent global design changes.
- `docs/WORK_REQUEST_TEMPLATE.md` - template for future user-to-AI requests.
